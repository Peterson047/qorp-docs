Modelo B · Embutido

Um grafo LangGraph com system prompt dinâmico por setor. O IAM resolve o isolamento via filtro de tools — sem múltiplos serviços.

Modelo B+ · Subgrafos
Agentes como Subgrafos
Quando setores tiverem lógicas de orquestração distintas, o agente vira um subgrafo dentro do core — sem nova infra, sem latência de rede.
Gatilho
Fluxo diferente por setor: human-in-the-loop, pipelines assíncronos, retry policies específicas
Modelo C · Híbrido
Serviços Externos via MCP
Agentes críticos viram serviços independentes chamados via MCP ou rota HTTP — escala e deployment separados do core.
Gatilho
Times distintos por agente, SLAs diferentes, necessidade de isolamento de falhas ou escala independente
Fluxo de uma Mensagem no Core
do canal até a resposta · Fase 1
1
Entrada pelo Canal
A mensagem chega pelo canal (WhatsApp via n8n, Web via API). O gateway valida o Bearer Token e identifica user_id + channel.
Payload
{ user_id: "usr_carlos_001", message: "gera proposta pro cliente X", channel: "whatsapp" }
2
IAM resolve o SecurityContext
PolicyEngine consulta cargo + hierarquia (Redis TTL 1h). Gera available_agents, deny_tags, deny_tools e output_mode pelo canal.
SecurityContext gerado
{ role: "Vendedor/SDR", available_agents: ["agente_comercial"],
deny_tags: ["admin","payments","financial"], output_mode: "condensed" }
3
Core monta o contexto do agente
O nó call_agent carrega o system_prompt do setor, recupera memória do usuário, e executa bind_tools() com apenas as tools permitidas pelo IAM.
O que o LLM enxerga
system_prompt: "Você é assistente do setor comercial..."
tools: [mcp_query, generate_proposal, knowledge_base_search,
        web_search, get_address_from_cep]
memory: últimas interações do Carlos
4
Resposta formatada por canal
O output_mode formata a resposta pelo canal. WhatsApp recebe texto condensado sem markdown; Web recebe resposta completa.
Output por canal
Web: "## Proposta para Cliente X\n\n**Produto:** ..."
WhatsApp: "Proposta pra X: Produto Y, R$ 1.200/mês. Envio PDF?"
5
Memória + Auditoria
consolidate_memory extrai fatos relevantes e salva na memória de longo prazo. audit_log registra tools usadas, tokens, latência e o SecurityContext ativo.
Adicionando um Novo Setor
sem reescrever o core · só configuração
Ex: adicionar RH — 4 passos
1 · IAM — novo scope
Cadastra scope "rh" e mapeia grupos Google do RH no Identity Registry
2 · Tools — novas tags
Cria tools do RH com tag "rh", sensitivity "restricted"
3 · Knowledge — novo scope
Ingere documentos com metadata.scope = "rh" na vector store
4 · Prompt — novo template
Adiciona system_prompts["rh"] no config. Pronto.

Princípios de Design
as regras que guiam cada decisão arquitetural
Config, não código
Novo setor = novo IAM scope + system_prompt + KB scope. O grafo LangGraph não muda. A lógica de orquestração é reutilizada.
Separar quando necessário
Agente vira subgrafo quando tem fluxo diferente. Vira serviço externo quando precisa escalar ou isolar falhas. Não antes.
IAM como fundação
O filtro de tools resolve o isolamento entre setores. A separação física de agentes é otimização operacional — não pré-requisito de segurança.
